# 成都中考 5+2 统招线预测器（V1.0 纯净版）

## 概述

预测成都"5+2"（青羊 / 锦江 / 金牛 / 武侯 / 成华 / 高新 / 天府）公办高中**统招**录取最低分。
机制为"三层主体 + 四补丁"，**系数锁死、不过拟合**：分段系数 5%/10%/12% 与节点 3000/5000/8000
每年不变，禁止用当年真实值回灌调参。深度参考见 `references/manual.md`。

## 何时使用

- 用户说"跑 202X 届 5+2 预测" → 自主搜数据 → 建三表 → 全量跑 40–50 所 → 建 R_final 内存库。
- 用户说"预测 [校] [年]" → 库含则四步直出；不含则搜该校后跑。
- 用户说"推荐 [分] [年] [区位/调剂偏好可选]" → 库反查冲稳保。
- 用户问民办 / 指标 / 调剂 / 项目班 / 郊县班 → **直接拒答**（硬约束 6）。
- **6 月底前**一分一段表与招生计划未发布，数据不全，不准硬跑。

## 硬约束（任何情况不可违反）

1. ❌ 禁止用当年真实值回灌调分段系数（5%/10%/12% 永久锁，节点 3000/5000/8000 永久不变）。
2. ❌ 禁止加第 5、第 6 补丁。
3. ❌ 禁止把 λ / δ / α=0.4 当拟合参数反推（仅查表取值）。
4. ❌ 分数输出只给**整数**，不保留小数。
5. ❌ 不准拿"总计划差"当 ΔE，必须只拆**统招**渠道（指标/调剂/项目班不计入）。
6. ❌ 不准预测民办 / 指标 / 调剂 / 项目班 / 郊县班，用户问了拒答。
7. ❌ 跨届不串数据，跑 2028 届时丢弃 2027 重搜。

## 工作流

### Phase 0 · 自主搜寻（用户说"跑 202X 届"后执行）

按 5 类数据搜集，来源优先级：
1. **考试院官网** → T-1 年 5+2 各校统招录取最低分 + 最低位次 R（约 40–50 所）
2. **考试院官网** → T 年一分一段表（分数→累计人数，全分段）
3. **教育局官网 /《招生计划合订本》** → T 年各校招生计划，**必须拆 4 渠道**：统招 / 指标到校 / 调剂 / 项目班
4. 同源 → T-1 年各校统招计划（算 ΔE = T统招 − T-1统招）
5. 同源 → T 年新增普通高中名单 + 单校统招计划（补丁 C 用）

可信度：考试院 / 教育局 > "成都教育发布" > 川观 / 成都商报（仅交叉印证）。双源冲突以考试院为准，差距大标 ⚠️。
- 单校缺 → 跳过，输出标 ⚠️「数据缺失」。
- 一分一段只有图 → OCR 或回用户要文字版，**不准估**。
- 招生计划只有总数没拆渠道 → 再搜该校《202X 招生简章》单行本补；补不到用总数估算，标 ⚠️「ΔE 为总数估算」。

### Phase 1 · 数据汇聚（内存三表）

- 表A_history：`school | R_T-1 | S_T-1`
- 表B_current：`score | cum_rank`（同时建反向插值 rank→score）
- 表C_plan：`school | regular_T-1 | regular_T | ΔE | quota_T | transfer_T | project_T`

**插值（线性，整数）**：`S = S_a + (R_q − R_a)/(R_b − R_a) × (S_b − S_a)`，结果 `int(round)`。
例：rank=468 落在 rank=402→649 与 rank=474→648 之间
→ 648 + (474−468)/(474−402) × (649−648) = 648.08 → **取整 648**。

### Phase 2 · 核心机制（系数锁死）

**三层主体**：
```
Step1 等值：表A 取 R → 表B 插值 → S_base
Step2 招生：ΔE = regular_T − regular_T-1，R₁ = R + ΔE
Step3 分流（系数固定，禁止拟合）：
    R₁ ≤ 3000        → D = 0
    3000 < R₁ ≤ 5000 → D = (R₁ − 3000) × 5%
    5000 < R₁ ≤ 8000 → D = 100 + (R₁ − 5000) × 10%
    R₁ > 8000         → D = 400 + (R₁ − 8000) × 12%
    R_final = R₁ + D
Step4 产出：表B 插值 R_final → S_pred（整数分）
```

**四补丁（触发式）**：
- **补丁A 首年校**（表A 无 T-1 记录）：`R_X = R_anchor × λ_leader × λ_location`
  - λ_leader：四七九本部 0.85–0.90 / 名校嫡系新校区 0.86–0.90 / 四七九分校 0.92–0.95 / 教科院系 0.94–0.97 / 区属 1.0
  - λ_location：五城核心 0.98 / 高新天府 1.0 / 近郊 1.02–1.05
- **补丁B 首年校次年**（表A 中该校 T-1 是首年）：`R = R_first × (1 + δ_cool)`
  - δ：七中领办 +8~12% / 石室树德领办 +5~8% / 教科院 +3~5%
- **补丁C 同段竞品稀释**：该校 ±50 分区间，T 年新增校统招合计 ≥150 → `R_final += 新增合计 × 0.4`
- **补丁D 异常校弃权**：改制 / 重大舆情 / 统招计划异动>30% 任一 → 标 🔴「模型失效」，置信 ±5~8 分，**建议人工**

### Phase 3 · 双向接口

**指令①：预测 [校] [年]** — 四步拆解 + 补丁 + 置信 ±1~2（异常校 ±5~8）。按此模板输出：
```
【XX中学 202X 统招线预测】
数据源：一分一段√ 招生计划√ 历史录取√（未√标⚠️）
基准位次(T-1)：R=xxxx（S_T-1=xxx）
├─ 等值：S_base=xxx
├─ 招生：ΔE=xx（统招 xx→xx）→ R₁=xxxx
├─ 分流：D=xx（XXXX段）→ R_final=xxxx
└─ 预测线：xxx 分（置信 ±1~2）
⚠️ 触发补丁X：（如有）
```

**指令②：推荐 [分] [年] [区位/调剂偏好可选]**：
1. 表B 把 S_user 插值得 R_user；2. 从 R_final 库反查；3. 梯度窗口：
   - 冲：[R_user−300, R_user−50]
   - 稳：[R_user−50, R_user+200]
   - 保：[R_user+200, R_user+800]
4. 按偏好过滤，每档 3–5 所。模板：
```
【S=xxx 分（R≈xxxx）· 5+2 推荐 · 202X】
🎯 冲：1.X（预xxx/R≈xx） 2.X 3.X
✅ 稳：1.X 2.X 3.X
🛡️ 保：1.X 2.X 3.X
⚠️ 模型 ±1 命中~67% ±2~90%，保档建议 R_user+500 起
```

## 使用自带工具

`scripts/predict.py` 是确定性计算器（系数与补丁在代码内同样锁死）。把三表 + 新增/异常元数据写入
一份 JSON（字段见 `assets/input_template.json`），运行即得全量预测线或推荐梯度，避免手算误差：

```bash
# 全量预测
python3 scripts/predict.py --input data.json --run-all
# 按分数推荐（可选 --pref 区位过滤）
python3 scripts/predict.py --input data.json --recommend 612 --year 2027 --pref 武侯
# 插值自检
python3 scripts/predict.py --selftest
```

用法：先按 Phase 0 搜齐数据 → 落盘为一份 JSON（结构同 `assets/input_template.json`）→ 跑引擎。
引擎输出即预测/推荐结果，再按上方模板向用户呈现（整数分、含补丁与置信标注）。

## 基线精度（向用户说明）

2026 届 5+2 老校 ~38 所回测：±1 分 ≈ 67%，±2 分 ≈ 90%。失控情形由补丁 A/B/C/D 覆盖，不在三层系数内。
