## 这是什么

`cd-zk-predict` 是一个给 CodeBuddy / Claude 类智能体使用的 **Skill（技能包）**。
它把"成都中考 5+2 统招线预测"这件事固化成一套可复用的工作流：

1. **Phase 0** — 自主搜寻 5 类数据（历史录取位次、一分一段表、分渠道招生计划、T-1 统招计划、新增校名单）。
2. **Phase 1** — 把数据汇聚成三张内存表（历史 / 一分一段 / 计划）。
3. **Phase 2** — 用「三层主体 + 四补丁」机制算出每所学校的预测线。
4. **Phase 3** — 提供「预测某校某年」「按分推荐冲稳保」两条双向指令。

最大的特点是 **系数永久锁死、禁止回灌调参**：分段系数 `5% / 10% / 12%`
与节点 `3000 / 5000 / 8000` 每年不变，因此不会过拟合、不会"用当年答案反推公式"。

## 核心机制

### 三层主体（系数锁死）

```
Step1 等值：表A 取去年最低位次 R → 一分一段表插值 → S_base
Step2 招生：ΔE = 今年统招 − 去年统招，R₁ = R + ΔE
Step3 分流（系数固定，禁止拟合）：
    R₁ ≤ 3000        → D = 0
    3000 < R₁ ≤ 5000 → D = (R₁ − 3000) × 5%
    5000 < R₁ ≤ 8000 → D = 100 + (R₁ − 5000) × 10%
    R₁ > 8000         → D = 400 + (R₁ − 8000) × 12%
    R_final = R₁ + D
Step4 产出：一分一段表插值 R_final → S_pred（整数分）
```

> 节点 `3000 / 5000 / 8000` 与系数 `5% / 10% / 12%` 永久锁死，禁止用当年真实值回灌。

### 四补丁（触发式）

| 补丁 | 触发条件 | 处理 |
|------|----------|------|
| **A 首年校** | 表A 无 T-1 记录 | 锚校法：`R_X = R_anchor × λ_leader × λ_location` |
| **B 首年校次年** | 表A 中该校 T-1 是首年 | `R = R_first × (1 + δ_cool)` |
| **C 同段竞品稀释** | 该校 ±50 分内，T 年新增校统招合计 ≥150 | `R_final += 新增合计 × 0.4` |
| **D 异常校弃权** | 改制 / 重大舆情 / 统招计划异动>30% | 标 🔴「模型失效」，置信 ±5~8，建议人工 |

**λ / δ / α=0.4 仅作查表取值，禁止反向拟合。**

## 安装

### 方式一：作为 CodeBuddy 用户技能

```bash
# 克隆或下载本仓库后，把 cd-zk-predict 目录放到用户技能目录：
#   Linux / macOS: ~/.codebuddy/skills/cd-zk-predict/
#   Windows:       %USERPROFILE%\.codebuddy\skills\cd-zk-predict\
```

目录结构必须为：

```
cd-zk-predict/
├── SKILL.md
├── scripts/predict.py
├── references/manual.md
└── assets/input_template.json
```

放好后重启 / 刷新 CodeBuddy，对助手说"跑 2027 届 5+2 预测"即可触发。

### 方式二：手动拼装

把仓库内 4 个文件按上表结构放到技能目录即可。

## 使用

### 命令式接口

`scripts/predict.py` 是确定性计算引擎（系数与补丁在代码内同样锁死）。
先把三张表 + 新增 / 异常元数据写入一份 JSON（字段见 `assets/input_template.json`），再运行：

```bash
# 全量预测
python3 scripts/predict.py --input data.json --run-all

# 按分数推荐（可选 --pref 区位过滤）
python3 scripts/predict.py --input data.json --recommend 612 --year 2027 --pref 武侯

# 插值自检
python3 scripts/predict.py --selftest
```

### 对话式指令

| 用户指令 | 动作 |
|----------|------|
| `跑 2027 届 5+2 预测` | Phase0 搜 → Phase1 聚 → Phase2 全量跑 → 建 R_final 库 |
| `预测 川大附中 2027` | 库含则四步直出；不含则搜该校后跑 |
| `推荐 612 分 2027 武侯` | 库反查 → 冲稳保，按"武侯"过滤 |

**预测模板输出示例：**

```
【XX中学 202X 统招线预测】
数据源：一分一段√ 招生计划√ 历史录取√（未√标⚠️）
基准位次(T-1)：R=xxxx（S_T-1=xxx）
├─ 等值：S_base=xxx
├─ 招生：ΔE=xx（统招 xx→xx）→ R₁=xxxx
├─ 分流：D=xx（XXXX段）→ R_final=xxxx
└─ 预测线：xxx 分（置信 ±1~2）
⚠️ 触发补丁X：（如有）
```

**推荐模板输出示例：**

```
【S=xxx 分（R≈xxxx）· 5+2 推荐 · 202X】
🎯 冲：1.X（预xxx/R≈xx） 2.X 3.X
✅ 稳：1.X 2.X 3.X
🛡️ 保：1.X 2.X 3.X
⚠️ 模型 ±1 命中~67% ±2~90%，保档建议 R_user+500 起
```

## 硬约束

技能内置 7 条不可违反的硬约束：

1. ❌ 禁止用当年真实值回灌调分段系数（`5%/10%/12%` 永久锁，节点 `3000/5000/8000` 永久不变）。
2. ❌ 禁止加第 5、第 6 补丁。
3. ❌ 禁止把 `λ / δ / α=0.4` 当拟合参数反推。
4. ❌ 分数输出只给**整数**，不保留小数。
5. ❌ 不准拿"总计划差"当 ΔE，必须只拆**统招**渠道。
6. ❌ 不准预测民办 / 指标 / 调剂 / 项目班 / 郊县班，用户问了拒答。
7. ❌ 跨届不串数据，跑 2028 届时丢弃 2027 重搜。
